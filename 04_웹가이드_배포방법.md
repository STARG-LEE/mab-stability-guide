## ✅ 방법 A — GitHub Pages (추천 · 권한 공유 불필요)

### ➊ 파일 올리기 (웹에서 드래그&드롭, 제일 쉬움)
1. https://github.com/STARG-LEE/mab-causal-network-v2 접속 → 로그인
2. **Add file → Upload files** 클릭
3. 내 PC의 `mab_teaching_package/notebooks/` 에서 **`01_pipeline_practice.ipynb`, `02_draw_graph.ipynb`** 를 끌어다 놓기
   - (폴더 구조를 만들려면 파일명 앞에 `notebooks/` 를 붙여 입력하거나, 업로드 후 경로를 맞춰도 됩니다)
4. 다시 **Upload files** → `mab_teaching_package/guide/` 폴더의 **`index.html` 과 `img/` 안 4장**을 `guide/` 경로로 올리기
5. 아래 **Commit changes** 클릭

> 💻 git이 익숙하면: 세 폴더(`notebooks/`, `guide/`)를 복사해 넣고 `git add . && git commit -m "add guide" && git push` 하면 끝.

### ➋ GitHub Pages 켜기 (이미 켜져 있으면 생략)
- 저장소 **Settings → Pages → Build and deployment**
- Source: **Deploy from a branch**, Branch: **main / (root)** → **Save**
- 1~2분 뒤 `https://starg-lee.github.io/mab-causal-network-v2/guide/` 가 열립니다.

### ➌ 저장소가 공개(public)인지 확인 — ⭐가장 중요
- Colab "GitHub에서 열기" 링크는 **공개 저장소**일 때만 권한 없이 열립니다.
- Settings → 맨 아래 **Danger Zone → Change visibility** 가 *Public* 이면 OK.
- 이게 핵심이에요: **public이기만 하면, 학생들에게 따로 뷰어 권한을 줄 필요가 전혀 없습니다.** 링크만 공유하면 끝.

### ➍ (필요 시) 내 저장소에 맞게 버튼 주소 1곳만 수정
`guide/index.html` 안의 Colab 버튼 주소가 본인 저장소와 다르면 두 줄만 고치세요.
형식:
```
https://colab.research.google.com/github/<유저>/<레포>/blob/<브랜치>/notebooks/02_draw_graph.ipynb
https://colab.research.google.com/github/<유저>/<레포>/blob/<브랜치>/notebooks/01_pipeline_practice.ipynb
```
현재 값: 유저=`STARG-LEE`, 레포=`mab-causal-network-v2`, 브랜치=`main` (그대로면 수정 불필요).

**끝!** 이제 그림책 주소만 공유하면, 학생이 버튼을 눌러 Colab에서 바로 실행할 수 있습니다.

## 🟡 방법 B — 구글 드라이브 (대안 · 권한 수동 설정 필요)

GitHub 대신 드라이브에 노트북을 둘 수도 있지만, **말씀하신 그 권한 문제**가 있어 손이 더 갑니다.

1. 드라이브에 `.ipynb` 업로드 → 더블클릭하면 Colab으로 열림
2. Colab에서 **공유** 버튼 → **일반 액세스**를 **"링크가 있는 모든 사용자"** + 역할 **"뷰어"** 로 변경
   - ⚠️ 이 단계는 **노트북마다 수동**으로 해야 합니다(자동 안 됨). 비공개로 두면 남이 못 엽니다.
3. **링크 복사** → 그림책 `index.html`의 버튼 `href` 에 붙여넣기
   - 드라이브 링크 형식: `https://colab.research.google.com/drive/<파일ID>`
4. 노트북을 수정/교체하면 링크·권한을 다시 챙겨야 합니다.

> 결론: **방법 A(GitHub)** 가 권한 설정이 없어 훨씬 편하고, 업데이트도 push 한 번이면 끝입니다. 그래서 추천합니다.
